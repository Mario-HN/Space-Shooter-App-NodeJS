11) Mapeamento objeto-relacional (ORM) é uma técnica de desenvolvimento que permite
consultar e manipular dados de um banco de dados usando o paradigma de orientação a
objetos. O uso dessa técnica envolve a codificação de 3 tipos de arquivos: modelos,
migrations e seeders. Explique com suas próprias palavas qual a finalidade de cada um
desses códigos.


ORM (Object-Relational Mapping) é uma técnica usada para mapear objetos do software 
para registros em uma base de dados relacional. Em ORM, os conceitos do mundo real 
são representados como objetos no software, e esses objetos são armazenados e recuperados 
da base de dados como registros.

Em um framework de ORM, os seguintes conceitos são comumente usados:

-Models: São as representações de objetos do mundo real no software. Em ORM, 
cada model é associado a uma tabela na base de dados e representa um tipo de 
registro que pode ser armazenado e recuperado da base de dados. Cada model tem 
atributos que correspondem a colunas na tabela da base de dados.

-Seeders: São scripts que são executados para preencher a base de dados com dados 
iniciais. Seeders são usados para criar dados fictícios para testes ou para fornecer 
dados iniciais para uma aplicação recém-instalada.

-Migrations: São scripts que são executados para modificar a estrutura da base de 
dados, como adicionar ou remover tabelas ou colunas. As migrations são usadas para 
manter a consistência da estrutura da base de dados ao longo do tempo, à medida que 
a aplicação é desenvolvida e evolui.
